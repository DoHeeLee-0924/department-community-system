# Department Community System

## 1. 프로젝트 개요

**Department Community System**은 학과 구성원들이 수업, 과제, 팀플, 공지, 취업 관련 정보를 효율적으로 공유할 수 있도록 설계한 학과 정보공유 커뮤니티 관리 시스템입니다.

기존 대학 커뮤니티가 자유게시판, 비밀게시판 등 게시판 단위로 정보를 분류하는 것과 달리, 본 시스템은 게시글 작성 시 반드시 카테고리를 선택하도록 하여 정보를 유형별로 구조화합니다. 또한 사용자 등급과 게시글 반응 데이터를 기반으로 정보 신뢰도 점수를 계산하여, 신뢰도 높은 게시글이 우선적으로 노출되도록 설계했습니다.

---

## 2. 주요 기능

* 회원가입 및 로그인
* 게시글 작성
* 게시글 카테고리 필수 선택
* 카테고리별 게시글 조회
* 게시글 상세 조회
* 댓글 작성
* 좋아요 기능
* 게시글 및 댓글 신고
* 사용자 등급 자동 산정
* 정보 신뢰도 점수 기반 게시글 정렬
* 관리자 전용 신고 관리 페이지
* 관리자 게시글 및 댓글 삭제 기능

---

## 3. 게시글 카테고리

게시글 작성 시 아래 카테고리 중 하나를 반드시 선택해야 합니다.

* 인사방
* 수업
* 과제
* 팀플
* 공지
* 취업

신입 등급 사용자는 일반 게시글 상세 내용을 볼 수 없지만, 인사방 게시글은 조회 및 작성할 수 있습니다.

---

## 4. 사용자 등급 체계

본 시스템은 네이버 카페와 유사한 등급 구조를 적용했습니다. 회원가입 시 모든 사용자는 기본적으로 `신입` 등급으로 시작하며, 활동 조건을 만족하면 자동으로 등급이 상승합니다.

| 등급  | 조건                     | 주요 권한                     |
| --- | ---------------------- | ------------------------- |
| 신입  | 가입 직후                  | 인사방 조회 및 글쓰기 가능           |
| 일반  | 게시글 1개 이상              | 게시글 상세 조회, 댓글, 좋아요, 신고 가능 |
| 우수  | 게시글 5개 이상 + 댓글 5개 이상   | 신뢰도 점수 가중치 상승             |
| 최우수 | 게시글 10개 이상 + 댓글 10개 이상 | 신뢰도 점수 추가 가중치             |
| 관리자 | 고정 계정                  | 전체 게시글·댓글 삭제, 신고 목록 조회    |

---

## 5. 관리자 계정

관리자는 시스템에 고정된 계정으로 로그인할 수 있습니다.

```text
ID: qwer
Password: 1234
```

관리자는 모든 게시글과 댓글을 삭제할 수 있으며, 사용자가 신고한 게시글과 댓글을 관리자 페이지에서 확인할 수 있습니다. 단, 관리자는 게시글과 댓글을 수정할 수 없고 삭제만 가능합니다.

---

## 6. 정보 신뢰도 점수 로직

본 시스템은 게시글을 단순 최신순이나 좋아요 수 기준으로만 보여주지 않고, 사용자 등급과 게시글 반응도를 함께 반영하여 정보 신뢰도 점수를 계산합니다.

```text
정보 신뢰도 점수 =
사용자 등급 가중치 × 10
+ 좋아요 수 × 2
+ 댓글 수 × 1
+ 조회수 × 0.1
```

사용자 등급별 가중치는 다음과 같습니다.

| 등급  | 가중치 |
| --- | --: |
| 신입  | 0.5 |
| 일반  | 1.0 |
| 우수  | 2.0 |
| 최우수 | 3.0 |
| 관리자 | 3.0 |

정보 신뢰도 점수가 높은 게시글은 홈 화면에서 우선적으로 노출됩니다.

## 7. 신고 기능

일반 등급 이상의 사용자는 게시글과 댓글을 신고할 수 있습니다.

신고 시에는 반드시 신고 사유를 입력해야 하며, 신고된 게시글과 댓글은 관리자 전용 페이지에서 확인할 수 있습니다.

## 12. 기대효과

본 시스템을 통해 학과 구성원은 수업, 과제, 팀플, 공지, 취업 등 필요한 정보를 카테고리별로 빠르게 탐색할 수 있습니다. 또한 사용자 등급과 게시글 반응도를 기반으로 신뢰도 높은 정보가 우선적으로 노출되므로, 중요한 학과 정보가 일반 게시글 속에 묻히는 문제를 완화할 수 있습니다.

관리자는 신고된 게시글과 댓글을 확인하고 삭제할 수 있어, 학과 커뮤니티를 보다 체계적으로 관리할 수 있습니다.
